## 🔧 Backend Deployment (Render)

### Step 1: Prepare MongoDB

**Option A: MongoDB Atlas (Recommended)**
```
1. Sign up: https://www.mongodb.com/cloud/atlas
2. Create free cluster
3. Add IP whitelist: 0.0.0.0/0
4. Get connection string: mongodb+srv://user:pass@cluster.mongodb.net/dbname
```

**Option B: Local MongoDB**
```
For production, use managed MongoDB only. Don't use local.
```

### Step 2: Get API Keys

**OpenAI API Key:**
```
1. https://platform.openai.com/api-keys
2. Create new key
3. Save securely
```

**Generate JWT Secret:**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Step 3: Deploy to Render

1. **Go to https://render.com**
2. **Sign up/Log in with GitHub**
3. **Create New → Web Service**
4. **Connect Repository**
   - Find: `Task2-AI-Based-Text-summarization-Platform-Labmentix`
   - Connect it
5. **Service Settings**
   ```
   Name: ai-summarization-server
   Environment: Node
   Region: (closest to users)
   Branch: main
   Root Directory: server
   Build Command: npm install
   Start Command: node index.js
   ```
6. **Add Environment Variables**
   ```
   NODE_ENV → production
   PORT → 5000
   MONGODB_URI → your-mongodb-connection-string
   OPENAI_API_KEY → sk-your-key
   JWT_SECRET → your-generated-secret
   CLIENT_URL → https://your-vercel-url.vercel.app
   LOG_LEVEL → info
   ```
7. **Click Deploy**

**⏳ Wait for deployment to complete**

### Step 4: Save Backend URL

From Render dashboard:
- Navigate to your service
- Copy the URL (e.g., `https://ai-summarization-server.onrender.app`)

## 🎨 Frontend Deployment (Vercel)

### Step 1: Prepare Frontend

The `vercel.json` at root is already configured. No manual changes needed.

### Step 2: Deploy to Vercel

1. **Go to https://vercel.com**
2. **Sign up/Log in with GitHub**
3. **Add New Project**
4. **Import Repository**
   - Find: `Task2-AI-Based-Text-summarization-Platform-Labmentix`
   - Select it
5. **Configure Project**
   - Framework: Auto-detect (will detect Vite from root `vercel.json`)
   - Root Directory: `client` (auto-detected from `vercel.json`)
   - Build Command: Auto (from `vercel.json`)
   - Output Directory: `client/dist` (auto-detected)
6. **Add Environment Variables**
   ```
   VITE_API_URL → https://your-render-backend-url.onrender.app
   ```
   Example: `https://ai-summarization-server.onrender.app`
7. **Click Deploy**

**⏳ Wait for deployment to complete**

### Step 3: Save Frontend URL

From Vercel dashboard:
- Navigate to your project
- Copy the domain (e.g., `https://your-app.vercel.app`)

## 🐛 Troubleshooting

### Vercel Build Fails

**Error: "vite build exited with code 127"**

This was happening because Vercel was trying to build from root.

✅ **Fixed by:**
- Creating `vercel.json` at root level with correct configuration
- Setting `Root Directory: client`
- Setting correct build command

If still failing:
1. Check Vercel deployment logs
2. Ensure `client/package.json` has all dependencies
3. Clear Vercel cache: Settings → Advanced → Clear Cache → Re-deploy

### Frontend Can't Connect to Backend

**Symptoms:** API calls fail, blank page, or 500 errors

**Check:**
1. Is backend deployed? Test: `curl https://your-backend.onrender.app/health`
2. Is `VITE_API_URL` set correctly in Vercel?
3. Are CORS settings correct in backend?

```bash
# In Render backend Settings → Environment
CLIENT_URL=https://your-vercel-url.vercel.app
```

### Backend Won't Start

**Check logs in Render:**
1. Dashboard → Your service → Logs
2. Look for error messages
3. Common issues:
   - Missing MongoDB connection string
   - Invalid OpenAI API key
   - Wrong port configuration

### Database Connection Fails

**If using MongoDB Atlas:**
1. Check connection string format
2. Verify IP whitelist includes `0.0.0.0/0` (or your Render IP)
3. Verify database name is correct
